# 高德地图API Key类型调整指南

## 一、问题分析

当前前端网页无法正确显示地图，主要原因是使用了**Web服务类API Key**，而前端网页地图显示需要**Web端(JS API)**类型的Key。

### 不同API Key类型的区别：
- **Web服务类Key**：用于服务器端调用高德地图Web服务API，如地理编码、路径规划等
- **Web端(JS API) Key**：专门用于前端网页调用高德地图JavaScript API，支持地图显示、交互等功能

## 二、登录高德地图开放平台操作指南

### 步骤1：访问高德地图开放平台
打开浏览器，访问高德地图开放平台：[https://lbs.amap.com/](https://lbs.amap.com/)

### 步骤2：登录开发者账号
- 点击右上角的「登录」按钮
- 使用已有的高德地图开发者账号登录
- 如果没有账号，点击「注册」创建新账号

### 步骤3：进入控制台
登录成功后，点击右上角的「控制台」进入开发者控制台

## 三、创建或切换为正确Key类型

### 选项A：创建新的Web端(JS API) Key

#### 3.1 进入应用管理
1. 在控制台左侧菜单中，点击「应用管理」→「我的应用」

#### 3.2 创建新应用
1. 点击页面右上角的「创建新应用」按钮
2. 填写应用名称（如：校园物流系统）
3. 选择应用类型（如：Web端(JS API)）
4. 填写应用描述（可选）
5. 点击「创建」按钮

#### 3.3 添加Web端(JS API) Key
1. 在刚创建的应用下，点击「添加」按钮
2. 在「Key名称」字段填写Key的名称（如：校园物流系统-前端地图）
3. 在「服务平台」下拉菜单中，**选择「Web端(JS API)」**（这是关键步骤！）
4. **设置安全密钥**（重要）：
   - 安全密钥是为了提高API调用的安全性
   - 点击「设置」按钮，按照提示完成安全密钥配置
5. **设置域名白名单**（可选但推荐）：
   - 点击「设置域名白名单」
   - 添加允许使用该Key的域名，如：localhost, your-domain.com等
   - 开发环境可以添加：localhost, 127.0.0.1
6. 点击「提交」按钮，完成Key的创建

### 选项B：修改现有Key的类型（如果允许）

1. 在「我的应用」页面，找到当前使用的Web服务类Key
2. 点击Key右侧的「编辑」按钮
3. 在「服务平台」下拉菜单中，将「Web服务」改为「Web端(JS API)」
4. 按照选项A中的步骤4-5设置安全密钥和域名白名单
5. 点击「提交」按钮，完成Key类型的修改

**注意**：部分旧版Key可能不支持修改服务平台，此时需要使用选项A创建新Key。

## 四、新Key的配置参数要求

### 4.1 必须配置项
- **服务平台**：必须选择「Web端(JS API)」
- **安全密钥**：必须配置，用于安全调用API

### 4.2 推荐配置项
- **域名白名单**：添加允许使用该Key的域名，提高安全性
- **Key名称**：使用清晰的名称，便于管理

## 五、前端代码中替换API Key

### 5.1 环境变量配置（推荐）

#### 步骤1：打开环境配置文件
1. 在项目中找到`.env`文件（如果没有，复制`.env.example`并重命名为`.env`）
2. 文件路径：`d:\TRAE\大创项目\大创\frontend\.env`

#### 步骤2：修改API Key配置
1. 找到`VITE_AMAP_KEY`配置项
2. 将其值替换为新创建的Web端(JS API) Key
3. 保存文件

示例：
```
# 高德地图API密钥 - Web端(JS API)类型
VITE_AMAP_KEY=你的新Web端API Key

# API基础URL
VITE_API_BASE_URL=http://localhost:5000/api

# 应用标题
VITE_APP_TITLE=校园物流系统

# 环境模式
VITE_ENV=development
```

### 5.2 直接修改代码（不推荐，仅临时测试使用）

如果需要临时测试，可以直接修改代码中的API Key：

1. 打开`AmapMap.vue`文件
2. 文件路径：`d:\TRAE\大创项目\大创\frontend\src\components\AmapMap.vue`
3. 找到以下代码行（大约第112行）：
   ```typescript
   const currentKey = import.meta.env.VITE_AMAP_KEY || '011b30e3bc887023f886c08cc71a319c'
   ```
4. 将默认值替换为新的Web端(JS API) Key

**注意**：直接修改代码中的默认值不是最佳实践，建议使用环境变量配置。

## 七、常见问题及解决方案

### 问题1：地图只显示网格，不显示地图内容
**原因**：API Key类型不正确或配置有误
**解决方案**：
- 确保使用的是Web端(JS API)类型的Key
- 检查Key的安全密钥配置是否正确
- 检查域名白名单是否包含当前使用的域名

### 问题2：控制台出现「INVALID_USER_KEY」错误
**原因**：API Key无效或类型不匹配
**解决方案**：
- 确认使用的是Web端(JS API)类型的Key
- 检查Key是否被禁用
- 检查Key的安全密钥是否配置正确

### 问题3：控制台出现「REFERER_NOT_ALLOWED」错误
**原因**：当前域名不在Key的域名白名单中
**解决方案**：
- 在高德地图控制台中，将当前域名添加到Key的域名白名单中
- 开发环境可以添加：localhost, 127.0.0.1

## 八、注意事项

1. **安全密钥必须配置**：新创建的Web端(JS API) Key必须配置安全密钥，否则地图无法正常显示
2. **域名白名单设置**：为了安全起见，建议只添加必要的域名到白名单
3. **Key的保护**：不要将API Key直接暴露在代码仓库中，使用环境变量配置
4. **定期更新Key**：定期更换API Key，提高系统安全性
5. **查看使用量**：在控制台中可以查看Key的使用量，避免超出免费额度
