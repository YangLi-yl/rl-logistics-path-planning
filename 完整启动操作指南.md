# 校园物流配送系统 - 完整启动操作指南

## 1. 电脑开机后的系统准备工作

### 1.1 系统环境检查
1. **开机登录**：使用您的Windows账户登录系统
2. **网络连接**：确保电脑已连接到互联网（用于验证服务和可能的依赖更新）
3. **系统权限**：确保您有足够的权限访问项目目录和运行命令

### 1.2 开发工具准备
1. **命令行工具**：按下 `Win + R`，输入 `powershell`，然后按 `Enter` 打开PowerShell
2. **项目目录**：导航到项目根目录
   ```powershell
   cd d:\TRAE\大创项目\大创
   ```
3. **环境变量检查**：确保Python和Node.js已正确配置到系统环境变量中
   ```powershell
   python --version
   node --version
   npm --version
   ```
   - 预期输出：
     ```
     Python 3.14.2
     v25.2.1
     11.6.2
     ```

## 2. 后端服务启动流程

### 2.1 环境检查
1. **进入后端目录**：
   ```powershell
   cd d:\TRAE\大创项目\大创\backend
   ```

2. **检查虚拟环境**：
   ```powershell
   Get-ChildItem -Name .venv -ErrorAction SilentlyContinue
   ```
   - 如果返回 `.venv`，表示虚拟环境已存在；否则需要创建

### 2.2 虚拟环境创建与激活
1. **创建虚拟环境**（如果不存在）：
   ```powershell
   python -m venv .venv
   ```

2. **激活虚拟环境**：
   ```powershell
   .venv\Scripts\activate
   ```
   - 激活后，命令行提示符前会显示 `(.venv)`

### 2.3 依赖安装
1. **安装核心依赖**：
   ```powershell
   # 安装FastAPI和相关依赖
   pip install fastapi uvicorn pydantic[email]
   
   # 安装强化学习相关库
   pip install stable-baselines3 gymnasium numpy
   
   # 安装其他依赖
   pip install uuid sqlite3 datetime json traceback
   ```

2. **验证安装**：
   ```powershell
   pip list | Select-Object -First 10
   ```
   - 确认已安装上述关键依赖

### 2.4 模型文件检查
1. **检查模型文件**：
   ```powershell
   cd ..\model
   Get-ChildItem models\campus_logistics_ppo.zip -ErrorAction SilentlyContinue
   ```
   - 如果模型文件不存在，从训练检查点复制：
     ```powershell
     copy train\models\checkpoints\campus_ppo_200000_steps.zip models\campus_logistics_ppo.zip
     ```

### 2.5 启动后端服务
1. **返回后端目录**：
   ```powershell
   cd ..\backend
   ```

2. **启动服务**：
   ```powershell
   python -m uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
   ```
   - 参数说明：
     - `--host 0.0.0.0`：服务监听所有网络接口
     - `--port 8000`：服务运行在8000端口
     - `--reload`：代码修改时自动重启服务（开发模式）

3. **等待服务启动**：
   - 当看到以下信息时，表示服务已启动成功：
     ```
     INFO:     Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
     INFO:     Started reloader process using WatchFiles
     ```

### 2.6 后端服务验证
1. **打开新的PowerShell窗口**（保持服务窗口运行）
2. **验证健康检查接口**：
   ```powershell
   Invoke-WebRequest -Uri http://localhost:8000/api/health -Method GET -UseBasicParsing
   ```
   - 预期输出：
     ```
     StatusCode        : 200
     Content           : {"status":"healthy","timestamp":"2026-01-18T10:00:00.000000"}
     ```

3. **验证根路径**：
   ```powershell
   Invoke-WebRequest -Uri http://localhost:8000/ -Method GET -UseBasicParsing
   ```
   - 预期输出：应用基本信息的JSON响应

## 3. 前端应用启动流程

### 3.1 环境检查
1. **进入前端目录**：
   ```powershell
   cd d:\TRAE\大创项目\大创\frontend
   ```

2. **检查Node.js和npm版本**：
   ```powershell
   node --version
   npm --version
   ```
   - 预期输出：
     ```
     v25.2.1
     11.6.2
     ```

### 3.2 依赖安装
1. **安装前端依赖**：
   ```powershell
   npm install
   ```
   - 此过程可能需要几分钟时间，取决于网络速度
   - 等待显示 `added X packages in Ys` 表示安装完成

2. **验证安装**：
   ```powershell
   npm list --depth=0 | Select-Object -First 10
   ```
   - 确认已安装Vue、Element Plus等关键依赖

### 3.3 配置高德地图API密钥
1. **检查配置文件**：
   ```powershell
   Get-ChildItem src\envs\AMAP_KEY.ts -ErrorAction SilentlyContinue
   ```

2. **如果文件不存在，创建配置**：
   ```powershell
   New-Item -Path src\envs -Name AMAP_KEY.ts -Force
   Set-Content -Path src\envs\AMAP_KEY.ts -Value "export const AMAP_KEY = 'your_amap_key_here';"
   ```
   - 请将 `your_amap_key_here` 替换为您的高德地图API密钥

### 3.4 启动前端应用
1. **启动开发服务器**：
   ```powershell
   npm run dev
   ```

2. **等待应用启动**：
   - 当看到以下信息时，表示前端服务已启动成功：
     ```
     VITE v5.0.10  ready in 1234 ms
     
     ➜  Local:   http://localhost:5173/
     ➜  Network: use --host to expose
     ```

### 3.5 前端服务验证
1. **检查服务状态**：
   - 前端服务默认运行在 `http://localhost:5173/`
   - 不要关闭此PowerShell窗口，保持服务运行

## 4. 验证应用界面成功显示

### 4.1 访问应用界面
1. **打开浏览器**：使用Chrome、Firefox或Edge浏览器
2. **输入前端地址**：在地址栏输入 `http://localhost:5173/`
3. **按下Enter键**：等待页面加载

### 4.2 确认界面元素
成功加载的应用界面应包含以下元素：

1. **顶部导航栏**：
   - 系统名称："校园物流配送系统"
   - 导航菜单：路径规划、历史记录等

2. **地图区域**：
   - 高德地图显示
   - 地图控件（缩放、定位等）
   - 地图中心默认显示校园区域

3. **左侧控制面板**：
   - 起点设置：可选择或输入起点坐标
   - 配送点列表：可添加、编辑和删除配送点
   - 规划参数配置：算法选择、车型选择等

4. **底部状态栏**：
   - 服务状态：显示后端服务连接状态
   - 系统版本信息

### 4.3 功能验证
1. **服务连接验证**：
   - 检查页面底部状态栏，确认"后端服务连接正常"

2. **路径规划测试**：
   - 点击"添加配送点"按钮
   - 在地图上点击两个不同位置
   - 点击"开始规划"按钮
   - 等待几秒后，地图上应显示规划的路径
   - 检查配送顺序和统计信息是否正确显示

## 5. 常见问题与解决方案

### 5.1 后端服务启动失败
- **端口被占用**：
  ```powershell
  netstat -ano | findstr :8000
  Stop-Process -Id <进程ID> -Force
  ```

- **依赖缺失**：
  ```powershell
  pip install -r requirements.txt  # 如果存在requirements.txt
  ```

### 5.2 前端服务启动失败
- **端口被占用**：
  ```powershell
  netstat -ano | findstr :5173
  Stop-Process -Id <进程ID> -Force
  ```

- **依赖冲突**：
  ```powershell
  rm -rf node_modules package-lock.json
  npm install
  ```

### 5.3 地图加载失败
- **API密钥错误**：检查 `src/envs/AMAP_KEY.ts` 中的密钥是否正确
- **网络连接问题**：确保电脑已连接到互联网

### 5.4 路径规划失败
- **后端服务未启动**：检查后端服务是否正常运行
- **模型文件缺失**：确认 `model/models/campus_logistics_ppo.zip` 文件存在
- **参数错误**：检查配送点信息是否完整和有效

## 6. 关闭服务步骤

### 6.1 关闭前端服务
1. 在运行前端服务的PowerShell窗口中，按下 `Ctrl + C`
2. 输入 `Y` 确认终止进程

### 6.2 关闭后端服务
1. 在运行后端服务的PowerShell窗口中，按下 `Ctrl + C`

### 6.3 退出虚拟环境
1. 在后端虚拟环境中，输入：
   ```powershell
   deactivate
   ```

2. 关闭所有PowerShell窗口

---

**注意事项**：
- 始终保持后端服务先于前端服务启动
- 在开发过程中，可使用 `--reload` 参数实现代码修改自动重启
- 定期备份数据库文件 `backend/upc_campus_logistics.db` 以防止数据丢失
- 生产环境部署时，请移除 `--reload` 参数并使用合适的服务器配置
