# 校园物流配送系统 - 用户使用指南

## 1. 项目概述与功能介绍

### 1.1 项目简介
校园物流配送系统是一个基于强化学习的智能路径规划系统，旨在优化校园内的物流配送路线，提高配送效率。系统采用前后端分离架构，后端使用FastAPI提供路径规划API，前端使用Vue3构建交互式界面，结合高德地图API实现可视化路径展示。

### 1.2 核心功能
- **智能路径规划**：支持贪心算法和强化学习模型两种路径规划方式
- **配送点管理**：可灵活添加、编辑和删除配送点信息
- **路径可视化**：在高德地图上直观展示规划路径和配送顺序
- **历史记录查询**：保存所有路径规划历史，支持查询和分析
- **多算法对比**：可比较不同算法的规划结果，包括距离、时间等指标

### 1.3 技术架构
- **后端框架**：FastAPI + SQLite
- **前端框架**：Vue3 + Element Plus + ECharts
- **地图服务**：高德地图API
- **路径规划算法**：贪心算法 + PPO强化学习模型
- **部署方式**：本地开发环境

## 3. 后端服务启动步骤

### 3.1 环境准备
1. **检查Python版本**：
   ```powershell
   python --version
   ```
   确保版本为3.8或以上。

2. **创建虚拟环境**（可选但推荐）：
   ```powershell
   cd d:\TRAE\大创项目\大创\backend
   python -m venv .venv
   .venv\Scripts\activate
   ```

3. **安装依赖**：
   ```powershell
   # 安装FastAPI和相关依赖
   pip install fastapi uvicorn pydantic[email]
   
   # 安装强化学习相关库
   pip install stable-baselines3 gymnasium numpy
   
   # 安装其他依赖
   pip install uuid sqlite3 datetime json traceback
   ```

### 3.2 数据库配置
系统使用SQLite数据库，无需手动配置。启动服务时会自动创建数据库文件和表结构。

- 数据库文件：`backend/upc_campus_logistics.db`
- 表结构：包含路径规划历史记录的所有信息

### 3.3 模型文件检查
确保强化学习模型文件存在：
- 模型路径：`model/models/campus_logistics_ppo.zip`
- 如果模型文件不存在，可从训练检查点复制：
  ```powershell
  cd d:\TRAE\大创项目\大创\model
  copy train/models/checkpoints/campus_ppo_200000_steps.zip models/campus_logistics_ppo.zip
  ```

### 3.4 启动后端服务

1. **进入后端目录**：
   ```powershell
   cd d:\TRAE\大创项目\大创\backend
   ```

2. **启动服务**：
   ```powershell
   python -m uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
   ```
   
   **参数说明**：
   - `--host 0.0.0.0`：表示服务将监听所有网络接口，这是服务端的配置参数
   - 实际访问时，请使用 `http://localhost:8000/` 或 `http://127.0.0.1:8000/`

3. **验证服务启动成功**：
   - 访问 `http://localhost:8000/api/health` 或 `http://127.0.0.1:8000/api/health`
   - 如果返回类似 `{"status":"healthy","timestamp":"2023-01-01T00:00:00.000000"}` 的结果，表示服务正常运行

### 3.5 常见问题排查

1. **端口被占用**：
   - 错误信息：`Error: [Errno 48] Address already in use`
   - 解决方法：使用其他端口启动服务，例如 `--port 8001`

2. **模型加载失败**：
   - 错误信息：`FileNotFoundError: No such file or directory: 'campus_logistics_ppo'`
   - 解决方法：检查模型文件路径是否正确，确保模型文件存在

3. **数据库错误**：
   - 错误信息：`sqlite3.OperationalError: table planning_history already exists`
   - 解决方法：删除数据库文件 `upc_campus_logistics.db` 后重新启动服务


### 4.2 安装依赖
1. **进入前端目录**：
   ```powershell
   cd d:\TRAE\大创项目\大创\frontend
   ```

1. 打开 `src/envs/AMAP_KEY.ts` 文件

   - 打开浏览器，访问命令行输出的地址（通常为 `http://localhost:5173`）
   - 如地址有变化，请以命令行输出为准

1. **依赖安装失败**：
   - 错误信息：`npm ERR! code EINTEGRITY`
   - 解决方法：清理npm缓存后重新安装
     ```powershell
     npm cache clean --force
     npm install
     ```

2. **端口被占用**：
   - 错误信息：`Error: Port 5173 is already in use`
   - 解决方法：修改 `vite.config.ts` 中的端口配置

3. **地图加载失败**：
   - 错误信息：`AMap is not defined`
   - 解决方法：检查高德地图API密钥是否正确配置

## 5. 基本功能操作说明

### 5.1 路径规划操作流程

1. **设置起点位置**：
   - 在地图上点击或搜索选择起点
   - 或手动输入经纬度坐标

2. **添加配送点**：
   - 点击"添加配送点"按钮
   - 输入配送点信息：
     - 位置：在地图上选择或输入经纬度
     - 货物重量：输入货物重量（kg）
     - 时间窗：设置配送时间范围（分钟）
   - 点击"确定"添加

3. **配置路径规划参数**：
   - **算法选择**：
     - `贪心算法`：基于最近距离的简单路径规划
     - `强化学习模型`：使用训练好的PPO模型进行智能规划
   - **车型选择**：选择配送车型（小型/中型/大型）
   - **模型选择**：如使用强化学习模型，可选择不同的模型版本

4. **执行路径规划**：
   - 点击"开始规划"按钮
   - 等待系统计算路径

5. **查看规划结果**：
   - **地图展示**：在地图上查看规划的路径和配送顺序
   - **配送顺序**：查看按最优顺序排列的配送点
   - **统计信息**：查看总距离、总时间等统计数据
   - **历史记录**：所有规划结果会自动保存到历史记录中

### 5.2 历史记录查询

1. **查看历史记录**：
   - 点击左侧菜单中的"历史记录"
   - 查看所有路径规划历史

2. **筛选和搜索**：
   - 可根据时间、算法、车型等条件筛选
   - 可搜索特定的规划记录

3. **查看详情**：
   - 点击历史记录条目，查看详细信息
   - 可在地图上重新展示路径

## 6. 常见问题解答与故障排除方法

### 6.1 后端服务问题

1. **API请求返回500错误**：
   - 检查后端服务是否正常运行
   - 查看后端日志，寻找具体错误信息
   - 常见原因：模型加载失败、数据库连接错误、输入参数格式错误

2. **健康检查接口无法访问**：
   - 检查服务是否启动
   - 检查防火墙设置，确保端口8000未被阻止

3. **模型推理速度慢**：
   - 原因：强化学习模型需要进行复杂计算
   - 解决方法：考虑使用贪心算法进行快速规划

### 6.2 前端应用问题

1. **页面加载缓慢**：
   - 检查网络连接
   - 清除浏览器缓存后重新加载

2. **地图无法显示**：
   - 检查高德地图API密钥是否正确
   - 检查网络连接是否正常
   - 确认浏览器是否支持WebGL

3. **添加配送点失败**：
   - 检查输入信息是否完整
   - 确认坐标格式是否正确（经度在前，纬度在后）
   - 检查时间窗设置是否合理（结束时间需大于开始时间）

### 6.3 数据和模型问题

1. **规划结果不合理**：
   - 检查输入的配送点信息是否正确
   - 尝试切换不同的算法
   - 对于强化学习模型，可考虑重新训练模型

2. **模型文件丢失**：
   - 从训练检查点复制模型文件：
     ```powershell
     cd d:\TRAE\大创项目\大创\model
     copy train/models/checkpoints/campus_ppo_200000_steps.zip models/campus_logistics_ppo.zip
     ```

3. **数据库损坏**：
   - 删除数据库文件 `backend/upc_campus_logistics.db`
   - 重新启动后端服务，系统会自动重建数据库

## 7. 注意事项与最佳实践

1. **数据安全**：
   - 系统使用SQLite数据库，数据存储在本地文件中
   - 定期备份数据库文件以防止数据丢失

2. **性能优化**：
   - 对于大量配送点（>20个），建议使用贪心算法
   - 合理设置配送点的时间窗，避免冲突

3. **系统维护**：
   - 定期更新依赖包以确保安全性
   - 监控系统日志，及时发现和解决问题
   - 根据实际需求调整强化学习模型的训练参数

4. **使用建议**：
   - 先使用贪心算法快速得到初步方案
   - 再使用强化学习模型获取更优方案
   - 比较不同算法的结果，选择最适合的方案
